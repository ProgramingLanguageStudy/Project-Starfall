### 기술 메모

- **CombatController 진실의 원천**: 전투 여부·전투 중인 적 목록을 CombatController가 소유, 필요한 곳에 주입. GameServices static 전투 제거로 주입 기반 구조.

- **이벤트 vs 직접 호출**: Enemy 전투 등록은 StateMachine → Enemy.NotifyCombatStateChanged → CombatController로 분리, StateMachine이 CombatController에 직접 의존하지 않음.

- **CompanionStateMachine 계층**: 행동 판단(Follow/Combat)과 액션(Idle/Attack/Dead) 분리. CompanionStateMachine = 행동 상태, CharacterStateMachine = 액션 상태.

- **IDamageable attacker 파라미터**: 데미지 원인자 전달로 전투 진입 판단(Enemy가 공격자 추적), OnDamaged 이벤트로 StateMachine 구독.

- **EnemyDetector 분리**: 탐지 로직을 별도 컴포넌트로 분리, 이벤트(OnCharacterDetected)로 어그로 연결. EnemyTeamData로 팀 구성·배치 데이터화.

- **전투 종료 플레이어 거리**: ChaseTarget 거리 대신 플레이어 10m 이탈 시 전투 종료. 동료 추적 시에도 플레이어 기준으로 일관성 유지.

- **PlaySaveCoordinator SerializeField**: Find 대신 인스펙터 할당. 의존성 명시·테스트 용이성 향상.
