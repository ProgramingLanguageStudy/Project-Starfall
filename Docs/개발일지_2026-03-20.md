### 기술 메모

- **부트**: Auth 초기화와 Data·Resource 로드는 동시에 시작하고, Save는 Auth `InitializeAsync` 직후 코루틴으로 시작. 완료는 `IsInitializeComplete`·`IsLoadComplete`·`DataManager.IsLoaded`·`ResourceManager.IsLoaded()`의 AND로만 `BootServicesReady`를 올림.
- **싱글톤 함정**: `GameManager.Instance`가 `FindFirstObjectByType`으로 `_instance`만 먼저 채우면, 기존 `if (_instance == null)` 안에서만 자식을 붙이는 `Awake`는 스킵되어 매니저가 전부 null이 됨 → 중복만 `Destroy`하고 항상 자식 연결.
- **세이브**: Firestore 백엔드는 Auth Phase가 `Pending`이 아닐 때 확정되므로, `LoadAsync`에서 Pending 타임아웃 대기 후 기본 세이브 폴백. `IsLoadComplete`는 부트 집계용.
- **인트로**: 타이틀 연출(`_titleSequenceFinished`) 전에는 로딩/로그인 패널을 올리지 않고, 이후에만 `BootServicesReady` 대기. 게스트는 이메일 없이 `ReadyGuest`/`InitFailed`에서 메인 패널로 진입(로컬 세이브 정책).

### 배운 점

- **Unity 생명주기**: `Awake` 순서는 보장되지 않으며, **정적 getter가 같은 프레임에서 먼저 실행**될 수 있어 `Awake` 안의 “첫 진입만 초기화” 조건은 `_instance == null`만으로는 부족할 수 있음. 싱글톤은 “내가 살아남는 인스턴스인지”와 “필드 연결”을 분리해 생각하는 편이 안전함.
- **비동기 부트**: 순차 `yield return` 체인은 읽기 쉽지만 I/O 시작이 늦어지므로, **병렬 시작 + 집계 플래그**로 바꾸면 연출·로딩 UI와 타임라인을 분리하기 쉬움.
