### 기술 메모

- **AIBrain**: 동료 전용 판단 레이어. CombatController.IsInCombat으로 Follow/TickCombat 분기. GetNearestEnemy로 타겟, 사거리 내에서 RequestAttack. Character API만 호출해 별도 상태머신 없이 플레이어와 동일 파이프라인 재사용.

- **IMovementHandler**: SetAsPlayer/SetAsCompanion 시 Handler 교체. Direction(입력 방향) vs Target(Transform 추적). Idle 전환 시 ClearDirectionIntent로 이전 입력 잔여 방지.

- **DeathHandler/DeadState 분리**: 죽음 시각·물리 처리는 DeathHandler에만. DeadState는 호출만. 사운드/이펙트 추가 시 한 곳만 수정.

- **포트폴리오 구조**: 핵심 6개(문제→해결→결과) + 전체 아키텍처(PlayScene 중심 연결도, 각 시스템 도식). AIBrain을 3.2로 분리해 강조.

### 배운 점

- **RequestIdle에서 Attack 리턴 차단**: 기존 `if (_currentState == _attackState) return;` 때문에 공격 종료 후 Idle 전환이 막혀 애니만 Idle로 가고 상태는 Attack에 머무름. 공격 완료 시 Idle로 가야 하므로 해당 조건 제거.

- **애니 MoveSpeed와 상태 분리**: Character.Update에서 Move 상태일 때만 CurrentMoveSpeed 전달. Idle/Attack/Dead에서는 0으로 해서 블렌드 트리가 Idle로 전환되도록 함.
