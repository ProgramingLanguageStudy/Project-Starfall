# 플래그·퀘스트·저장로드 구조 정리

> 이 문서는 "누가 뭘 들고 있는지", "저장/로드 시 무슨 일이 일어나는지"를 한눈에 이해하기 위한 정리입니다.

---

## 1. 런타임: 누가 뭘 들고 있는가

```
┌─────────────────────────────────────────────────────────────────┐
│  FlagSystem                                                      │
│  ─────────────────                                               │
│  모든 플래그 저장소 (key → int)                                    │
│  • first_talk_chief = 1                                          │
│  • quest_mushroom_gather_accepted = 1                             │
│  • quest_mushroom_gather_objectives_done = 1                      │
│  • quest_mushroom_gather_completed = 1                            │
│  • (킬 퀘스트: enemy_goblin_killed = 5 등 int도 가능)              │
└─────────────────────────────────────────────────────────────────┘
         ↑ 읽기                          ↑ 쓰기
    DialogueSelector              QuestController, 대화 종료 시

┌─────────────────────────────────────────────────────────────────┐
│  QuestSystem                                                     │
│  ─────────────────                                               │
│  진행 중 퀘스트 목록 + 진행도                                      │
│  • mushroom_gather: targetId=item_mushroom, currentAmount=7/10    │
│  • (퀘스트가 완료되면 목록에서 제거)                                │
└─────────────────────────────────────────────────────────────────┘
         ↑ 읽기                          ↑ 쓰기
    QuestPresenter(UI)              AcceptQuest, NotifyProgress, CompleteQuest

┌─────────────────────────────────────────────────────────────────┐
│  Inventory                                                       │
│  ─────────────────                                               │
│  슬롯별 아이템 + 개수                                              │
└─────────────────────────────────────────────────────────────────┘
```

**핵심**: FlagSystem과 QuestSystem은 **다른 정보**를 가짐.  
- FlagSystem: "수락했는지, 목표 달성했는지, 완료했는지" (예/아니오 또는 int)  
- QuestSystem: "어떤 퀘스트가 진행 중인지, 얼마나 했는지" (진행도 숫자)

---

## 2. 왜 플래그가 두 군데에?

### 대화 시스템의 입장
- 대화는 "이 플래그가 1이면 이 대화 보여줘" 방식으로 동작
- 대화 제작자는 **플래그 이름만** 알면 됨 (QuestSystem 몰라도 됨)
- 그래서 DialogueSelector는 `FlagSystem.GetFlag(key)`만 호출

### 퀘스트 시스템의 입장
- 퀘스트는 "몇 개 모았는지", "몇 마리 잡았는지" 같은 **진행도**가 필요
- 이건 QuestSystem이 들고 있음 (QuestModel.currentAmount)

### 동기화
- 퀘스트 이벤트 시 QuestController가 **양쪽**을 갱신:
  - 수락 → QuestSystem.AcceptQuest + FlagSystem.SetFlag(accepted)  
  - 목표 달성 → QuestSystem 진행도 갱신 + FlagSystem.SetFlag(objectives_done)  
  - 완료 → QuestSystem.CompleteQuest(제거) + FlagSystem.SetFlag(completed)

## 3. 저장 파일 구조 (JSON)

```json
{
  "squad": { ... },
  "flags": { "keys": ["first_talk_chief", "quest_mushroom_gather_completed"], "values": [1, 1] },
  "quests": { "entries": [{ "questId": "mushroom_gather", "targetId": "item_mushroom", "currentAmount": 7 }] },
  "inventory": { "slots": [...] }
}
```

| 필드 | 저장하는 것 | 담당 |
|------|-------------|------|
| flags | QuestCompleted, first_talk 등 (QuestAccepted/ObjectivesDone **제외**) | FlagSaveContributor |
| quests | 진행 중 퀘스트 목록 + 진행도 | QuestSaveContributor |
| inventory | 슬롯별 아이템 | InventorySaveContributor |

---

## 4. 저장 시 (Gather)

| Contributor | 하는 일 |
|-------------|----------|
| FlagSaveContributor | FlagSystem에서 **QuestAccepted, QuestObjectivesDone 제외**하고 나머지 저장 |
| QuestSaveContributor | QuestSystem.GetActiveQuests() → QuestSaveData. **수집형은 currentAmount=0 저장** (인벤토리가 진실의 원천). 처치/방문형은 저장. |
| InventorySaveContributor | Inventory.GetSlots() → InventorySaveData로 변환 |

**왜 Accepted/ObjectivesDone을 제외?**  
→ QuestSaveData에 "진행 중 퀘스트 목록"이 있으면, 로드 시 복원하면서 그걸로 유도 가능. 중복 저장 방지.

---

## 5. 로드 시 (Apply)

**순서**: Flag(1) → **Inventory(2)** → **Quest(3)**

| Contributor | 하는 일 |
|-------------|----------|
| FlagSaveContributor | FlagSaveData → FlagSystem에 복원 (QuestCompleted 등) |
| InventorySaveContributor | InventorySaveData → Inventory 복원 |
| QuestSaveContributor | QuestSaveData → QuestSystem 복원. **수집형은 Inventory.GetTotalCount(targetId)로 진행도 설정**. 처치/방문형은 entry.currentAmount 사용. + FlagSystem에 Accepted/ObjectivesDone 설정 |

**왜 Inventory가 Quest보다 먼저?**  
→ 수집형 퀘스트 진행도는 인벤토리에서 참조하므로, 인벤토리가 먼저 복원되어야 함.

---

## 6. 플로우 요약

```
[플레이 중]
  수락 → QuestSystem.AcceptQuest + FlagSystem.SetFlag(accepted)
  채집/처치 → QuestSystem.NotifyProgress + (목표 달성 시) FlagSystem.SetFlag(objectives_done)
  완료 → QuestSystem.CompleteQuest + FlagSystem.SetFlag(completed)

[저장]
  FlagSaveContributor: flags (QuestAccepted/ObjectivesDone 제외)
  QuestSaveContributor: quests (진행 중 목록 + 진행도)
  InventorySaveContributor: inventory

[로드]
  FlagSaveContributor: flags 복원
  QuestSaveContributor: quests 복원 + Accepted/ObjectivesDone 플래그 설정
  InventorySaveContributor: inventory 복원
```

---

## 7. 초기화 시 주의사항

**플래그만 초기화**하면 → QuestSystem은 그대로 → 불일치  
→ "완료" 대화가 안 보임 (accepted, objectives_done이 0)

**해결**: 테스트용 초기화 시 **플래그 + QuestSystem** 둘 다 초기화.

---

## 8. 현재 구조의 장단점

| 장점 | 단점 |
|------|------|
| 대화 제작자는 플래그만 알면 됨 | QuestSystem ↔ FlagSystem 동기화 필요 |
| QuestSaveData는 진행도만 저장 (중복 최소화) | 초기화 시 둘 다 건드려야 함 |

---

## 9. 대안 (참고용, 미구현)

- **FlagSystem이 Accepted/ObjectivesDone도 저장**: FlagSaveContributor에서 필터 제거. QuestSaveContributor는 플래그 설정 안 함. 대신 같은 정보가 두 군데에 있음.
- **QuestSaveData 제거**: Gather는 인벤토리에서, Kill은 FlagSystem int에서. → 진행도 복원 로직이 복잡해짐.
